Git

  • Lots of git notes to put here
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull: command to recieve a branches modifications onto my local computer
  • pwd: print working directory (displays current working directory ((use to confirm the directopry you are in))
  • Ls: shows a list of files in current directory
  • cd: Change directory (apply filename?)
  • mkdir: makes a new directory (or folder)
  • touch: creates a file inside of current directory (eg. touch newtxtfile.txt)
  • start: opens application or file (eg. start newtxtfile.txt)
  • code .: opens VScode (when your in applicable directory)
  • git commit -m "reference message goes here with quotations": pushes your version of code to github for merging (has not yet been merged)
  • git add -A: adds all files and changes to repo branch

    • Breakdown on how to push shiz to your repo n ...shiz

  • 1. git add -A: Adding files to the staging area of git ( once the files are fixed/created )
  • 2. git commit -m "notes of code change/bug fixes": changes have been staged, we can commit to our repository.
  • 3. Make sure our current main branch data is up-to-date BEFORE opening the PR
  • 4. git pull origin main:
  • 5. And now we push up all of our changes to the featured branch, to git hub
  • 6. git push origin feature/put-feature-name-here:
  • 7. go to your github repo, click pull requests tab and then the new pull request button
  • 8. make sure Base branch is main and Comparison branch is the new feature/bug fix
  • 9. add short description and title of changes made
  • 10. create pull request; confirm merge
  • 11. Now update your local side of the main branch to have the most updated, completed merge.
  • 12. git checkout main:
  • 13. git pull origin main:
